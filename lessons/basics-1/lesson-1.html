# Краткое описание языка C#

C# является одним из самых распространенных языков в России и во всем мире. Много вакансий, много реализованных приложений, которые нужно поддерживать, и большие перспиктивы развития языка.

# Почему С#?

  • Достаточно прост в изучении

  • Постоянная поддержка и доработка со стороны microsoft

# Знакомство с рабочим пространством

В правой части экрана Обозреватель решений содержит список проектов в решении

Структура решения и проектов в Обозревателе решений отличается от структуры в папке Windows, среда разработке отображает их в более удобном для разработки виде

  • Решение .sln - Набор проектов для решения определенных задач ползователя, представляет из себя полноценный продукт

  • Проект .csproj - Решает отдельную конкретную задачу продукта

Пример:

  • Социальная сеть VK - решение
  • Сайт, Панель для администраторов VK, Серверная часть, Вспомогательные библиотеки - отдельные проекты

В центре экрана код выбранного в Обозревателе решений файла

В нижней части экрана консоль с детализацией информации о ходе сборки и работы приложения (ошибки, предупреждения и т.д.)

В врхней части экрана кнопка для запуска проекта в режиме отладки

В режиме отладки Debug можно понять, в какой точке программы выполняется определенное действие

Также проект можно собрать в режиме Release, тогда мы получим готовое приложение, которое можно запустить на любом другом компьютере

В верхней панели переключить Debug на Release, в обозревателе решений правой кнопкой мыши нажать на проект и выбрать Собрать

В папке проекта появится готовый к запуску .exe файл

# Запуск первой программы

Любой проект на C# на платформе .NET запускается с файла Program.cs

Поменять код файла Program.cs на

using System;

namespace ConsoleApp1
{
    class Program
    {
        static void Main(string[] args) 
        {
            Console.WriteLine("Hello world");
            Console.ReadKey(); 
        }
    }
}

В открывшейся консоли появится запись

Hello world

Как это произошло? Код программы скопилировался в готовый для запуска на Windows .exe файл

Но не все ошибки выялвяются на этапе написание, часть из них может появиться при копиляции

# Вывод текста в консоль

Команды для вывода текста в консоль

Console.WriteLine("текст");
Console.Write("текст");

Разница между Console.Write и Console.WriteLine в том, что после Console.Write следующий вывод текста пойдет сразу же после предыдущего в одну строку, а после Console.WriteLine происходит перенос строки, и далее текс выводится с новой строки

Для наглядности можно запустить следующий набор инструкций

Console.Write("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world"); 
Console.ReadKey();

Текст передаваемый в в кавычках - является аргументом для инструкции или по другому входными данными

После вызова команд всегда ставится ;

# Комментарии

Проблема: мы написали код и

  1. Хотим чтобы часть временно не работала, но удалять код не будем, так как возможно он еще пригодится
  2. Хотим чтобы к части кода шли пояснения

Решают данную проблему комментарии

//Console.WriteLine("Закомментированный текст");
Console.WriteLine("Текст"); //вывод строки Текст в консоль
/** 
Console.Write("Закомментированный текст");
Console.WriteLine("Закомментированный текст");
*/
Console.ReadKey();
  • Комментарии в Visual Studio выделяются зеленым цветом
  • Комментарии игнорируются компилятором, то есть не попадает в сборку и не исполняется как инструкция

Есть 2 варианта комментариев

// однострочный
/**
многострочный
*/

Комбинации

Ctrl + k, c - для комментирования выделенного текста

Ctrl + k, u - для раскомментирования

# Переменные

Пример:

Console.WriteLine("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world");
Console.WriteLine("Hello world");  
Console.ReadKey();

Проблема: Что, если нам нужно поменять текст во всех 5 строках вывода, в случае большого проекта, где один и тот же текст используется в множестве мест, это становится затруднительным

Решение: использовать переменные

string greeting; // объявление переменной где string - тип данных переменной, а greeting - наименование
greeting ="Hello"; // инициализация переменной, где "Hello" - значение переменно
Console.WriteLine(greeting);
Console.WriteLine(greeting);
Console.WriteLine(greeting);
greeting ="Hello world"; // изменение переменной
Console.WriteLine(greeting);
Console.WriteLine(greeting);
string text ="Text"; // объявление переменной с инициализацией
Console.WriteLine(text);
Console.WriteLine(text);
Console.WriteLine(greeting);
Console.ReadKey();

Что такое переменные?

Это некие емкости данных с уникальным в рамках своего блока названием, содержимое которых мы можем менять

Переменные принято писать со строчной буквы

Перед объявлением переменных указывается тип данных К примеру string - строка

Для использования переменная должна быть проинициализирована

После объяления и инициализации переменной всегда ставится ;

# Константы

Рассмотрим пример:

string appName = "Telegram"; 
Console.Write("Добро пожаловать в ");
Console.Write(appName);

В данном случае appName - наименование приложения, и мы не хотим чтобы оно менялось

Тогда нужно использовать константу

Для appName добавим модификатор const

const string appName = "Telegram";  //модификатор const пишется всегда перед типом
Console.Write("Добро пожаловать в ");
Console.Write(appName);

Теперь, если попытаться изменить appName, студия покажет нам ошибку

Константы должны всегдя объявляться с инициализацией

После объяления и инициализации констант всегда ставится ;
